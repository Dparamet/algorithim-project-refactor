# ระบบจัดการคิวแลกเงิน (Queue-based Support Distribution)

เว็บแอปนี้เป็นระบบจัดการคิวรายชื่อผู้รับสิทธิ์ พร้อมคำนวณวงเงินตามสัดส่วน และแสดงผลการแจกเงินในแต่ละรอบ

> พัฒนาแบบ Frontend-only (HTML/CSS/JavaScript) ใช้งานได้ทันทีผ่านเบราว์เซอร์โดยไม่ต้องติดตั้งแพ็กเกจเพิ่ม

## ภาพรวมระบบ
ระบบนี้ช่วยจัดการรายชื่อบุคคลเพื่อรับเงินสนับสนุน โดยมี 3 ส่วนงานหลัก:

1. **จัดการรายชื่อ**
   - เพิ่มรายชื่อแบบกรอกเอง
   - สุ่มรายชื่อจำนวนมาก
   - ดู/แก้ไข/ลบรายบุคคล หรือ ลบทั้งหมด

2. **กำหนดงบประมาณ**
   - ใส่งบรวม
   - ใส่สัดส่วน 3 กลุ่มให้รวมกันเป็น 100%
   - ระบบคำนวณงบแต่ละกลุ่ม + ค่าเฉลี่ยต่อคน

3. **ติดตามผลการแจกเงิน**
   - แสดงผลรอบแจกเงินล่าสุด
   - สรุปจำนวนเงินแจกแล้ว/คงเหลือ
   - รายงานแยกตามกลุ่ม

---

## คุณสมบัติหลัก
- จัดลำดับคิวแบบอัตโนมัติด้วยเกณฑ์ **อายุมากก่อน** และถ้าอายุเท่ากันให้ **รายได้น้อยก่อน**
- ตรวจสอบความถูกต้องของข้อมูลงบประมาณ (ห้ามติดลบ, สัดส่วนต้องรวม 100%)
- คำนวณสัดส่วนวงเงินตามกลุ่ม: ผู้สูงอายุ / ผู้มีรายได้น้อย / เกษตรกร
- แสดงสถิติการแจกเงินและรายงานสรุปแยกกลุ่ม

---

## โครงสร้างไฟล์
- `index.html` — โครงสร้างหน้า UI ทั้งหมด (ฟอร์ม, หน้ารายชื่อ, หน้างบ, หน้าแจกเงิน, หน้ารายงาน)
- `style.css` — สไตล์ของระบบ (layout, card, list, สีปุ่ม, ส่วนแสดงสถิติ)
- `script.js` — ตรรกะหลักทั้งหมด (เพิ่มรายชื่อ, สุ่มข้อมูล, จัดลำดับคิว, คำนวณงบ, จำลองรอบแจกเงิน, รายงาน)

---

## วิธีใช้งานแบบเร็ว
1. เปิดไฟล์ `index.html` ในเบราว์เซอร์
2. เพิ่มรายชื่อด้วยการกรอกฟอร์ม หรือใช้ปุ่มสุ่มรายชื่อ
3. กด **กำหนดวงเงินและสัดส่วน** แล้วกรอกงบรวม + สัดส่วน 3 กลุ่มให้รวมเป็น 100%
4. กด **ดูข้อมูลการแจกเงิน** เพื่อดูผลการแจกในรอบนั้น
5. กด **ดูรายงานการแจกเงิน** เพื่อดูสรุปแยกตามกลุ่ม

---

## คู่มือการใช้งานทีละขั้น

### 1) เพิ่มข้อมูลผู้รับสิทธิ์
ที่หน้าหลัก (`manualForm`):
- กรอก ชื่อ-นามสกุล
- รหัสประชาชน
- อายุ
- รายได้ต่อเดือน
- เลือกกลุ่มเป้าหมาย
- กด **เพิ่มเข้าคิว**

> ทางลัด: ใช้ช่องจำนวนและปุ่ม **สุ่มข้อมูลรายชื่อบุคคล** เพื่อสร้างข้อมูลจำนวนมาก

### 2) ดูและจัดการรายชื่อ
กด **ดูรายชื่อบุคคล**
- ระบบจะแสดงรายชื่อเรียงตามลำดับความสำคัญ
- ปุ่ม **แก้ไข**: ดึงข้อมูลกลับไปหน้าแรกเพื่อแก้ แล้วบันทึกใหม่
- ปุ่ม **ลบ**: ลบรายชื่อเฉพาะคน
- ปุ่ม **ลบข้อมูลทั้งหมด**: ลบผู้ใช้ทั้งหมดในหน่วยความจำ

### 3) กำหนดงบประมาณและสัดส่วน
กด **กำหนดวงเงินและสัดส่วน**
- ใส่ **วงเงินรวม**
- ใส่เปอร์เซ็นต์ทั้ง 3 กลุ่ม
- กด **คำนวณการแบ่งเงิน**

ระบบจะแสดง:
- งบรวมต่อกลุ่ม
- จำนวนคนในกลุ่ม
- ค่าเฉลี่ยเงินต่อคน

### 4) ดูข้อมูลการแจกเงิน
กด **ดูข้อมูลการแจกเงิน**
- ระบบคำนวณผล “รอบแจกเงิน” ปัจจุบัน
- แสดงสถานะรายคนว่า **ได้รับแล้ว** หรือ **รอคิว**
- แสดงยอดเงินแจกแล้ว / คงเหลือ / รอบที่

### 5) ดูรายงานสรุป
กด **ดูรายงานการแจกเงิน**
- สรุปจำนวนคนและวงเงินที่ใช้จริงแยกตาม 3 กลุ่ม
- แสดงยอดรวมแจกแล้วและยอดคงเหลือ

## ระบบทำงานอย่างไร (Logic ภายใน)

### 1) โครงสร้างข้อมูลหลัก
ใน `script.js` มีตัวแปรสำคัญ:
- `users` — เก็บข้อมูลผู้ใช้งานทั้งหมด
- `distributionHistory` — เก็บประวัติผลการแจกเงินแต่ละรอบ
- `roundCounter` — เลขรอบแจกเงิน
- `FIXED_SUPPORT_AMOUNT = 100000` — จำนวนเงินคงที่ต่อคนต่อรอบ

### 2) การจัดลำดับคิว
ฟังก์ชัน `byPriority(a, b)` ใช้เรียงคิว:
1. อายุจากมากไปน้อย
2. ถ้าอายุเท่ากัน เรียงรายได้จากน้อยไปมาก

### 3) การคำนวณวงเงินตามสัดส่วน
`calculateBudgetDistribution(...)` จะ:
- อ่านจำนวนคนแต่ละกลุ่มจาก `getGroupCounts()`
- คำนวณงบกลุ่ม = `(งบรวม × เปอร์เซ็นต์กลุ่ม) / 100`
- คำนวณเฉลี่ยต่อคนของแต่ละกลุ่ม

### 4) การแจกเงินในรอบ
`calculateDistributionRound()` จะ:
- เรียงผู้ใช้ตามคิว
- แจกเงินทีละคนแบบคงที่ `100,000 บาท`
- ถ้างบคงเหลือไม่พอคนถัดไป จะเป็นสถานะ `รอคิว`
- ส่งกลับข้อมูลรอบ + ยอดแจกแล้ว + งบคงเหลือ

### 5) รายงาน
`generateReport()` จะอ่านผลจากรอบล่าสุดใน `distributionHistory` (หรือคำนวณรอบใหม่หากยังไม่เคยเปิดหน้าการแจก) แล้วสรุป:
- จำนวนผู้ได้รับจริงต่อกลุ่ม
- วงเงินที่ใช้จริงต่อกลุ่ม
- ยอดรวมแจกแล้วและยอดคงเหลือ

---

## กฎสำคัญของระบบ
- ข้อมูลงบประมาณต้องเป็นตัวเลขและห้ามติดลบ
- สัดส่วน 3 กลุ่มต้องรวมกันเป็น **100%** เท่านั้น
- การเรียงคิวมีผลต่อคนที่จะได้รับเงินก่อน
- เงินช่วยเหลือที่แจกจริงในรอบ ใช้ค่าคงที่ `100,000 บาท/คน`

---

## ข้อจำกัดที่ควรรู้
- ข้อมูลทั้งหมดเก็บในหน่วยความจำของหน้าเว็บ (ยังไม่บันทึกฐานข้อมูล)
- รีเฟรชหน้าเว็บแล้วข้อมูลจะหาย
- ระบบไม่มี backend และไม่มีระบบล็อกอิน
- แม้จะคำนวณงบตามสัดส่วนได้ แต่ตอน “แจกจริง” ใช้กฎเงินคงที่ต่อคน

---

## แนวทางทดสอบระบบ
แนะนำทดสอบกรณีต่อไปนี้:
1. เพิ่มข้อมูล 5–10 คนที่อายุ/รายได้ต่างกัน แล้วตรวจสอบลำดับคิว
2. กำหนดสัดส่วนรวมไม่เท่ากับ 100% แล้วดูว่าระบบแจ้งเตือนถูกต้อง
3. ตั้งงบรวมให้พอแจกแค่บางคน เพื่อตรวจสอบสถานะ `ได้รับแล้ว` / `รอคิว`
4. เปรียบเทียบยอดรวมในหน้าการแจกเงินกับหน้ารายงานว่าตรงกัน
